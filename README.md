# Travel Agent + Fact-Checker

Мультиагентная система для планирования путешествий. Принимает запрос вроде «Хочу в ноябре 2026 в Австралию» и выдаёт структурированный гид: официальные требования, реальный опыт путешественников, практические рекомендации и чеклист.

Построен на [LangGraph](https://github.com/langchain-ai/langgraph) + [OpenRouter](https://openrouter.ai/). Включает встроенный fact-checker, который проверяет итоговый ответ и инициирует перезапись при ошибках.

---

## Как работает

```
Запрос пользователя
        │
        ▼
  parse_travel_profile()
        │
        ├── есть незаполненные поля? → QUESTIONS_TO_USER (интерактивный диалог)
        │
        └── профиль полный ↓
                │
    ┌───────────┼───────────┐
    ▼           ▼           ▼
official_  reviews_   practical_
docs_sub   subagent   recs_sub
agent                 agent
    │           │           │
    └───────────┴───────────┘
                │
          aggregate_answer()
                │
                ▼
        run_fact_check_gate()
                │
                ├── APPROVED → финальный ответ
                │
                └── REWRITE → extract_focus_plan()
                              → перезапуск нужных субагентов
                              → aggregate_answer() (до 3 попыток)
```

### Три субагента (параллельно по логике, последовательно по коду)

| Субагент | Что ищет | Источники |
|---|---|---|
| `official_docs_subagent` | Официальные визовые/въездные требования | Сайты консульств, правительственных органов |
| `reviews_subagent` | Реальный опыт путешественников | Reddit, форумы, блоги (без TripAdvisor) |
| `practical_recs_subagent` | Что взять, купить, подготовить | Чеклисты, советы по технике, SIM, адаптерам |

Каждый субагент работает по одной схеме:
1. Генерирует seed-запросы под профиль поездки
2. Ищет через DuckDuckGo → парсит найденные URL
3. LLM решает: достаточно данных или нужен ещё один поисковый раунд (`need_more: true/false`)
4. Генерирует структурированный summary

### Fact-check gate

После сборки ответа запускается тот же LangGraph-граф, что и в базовом fact-checker агенте. Если ответ получает статус `REWRITE`:
- LLM анализирует критику и определяет, какие из трёх субагентов нужно перезапустить
- Субагенты получают `focus_claims` — конкретные утверждения для уточнённого поиска
- Собирается новый ответ (максимум 3 попытки)
